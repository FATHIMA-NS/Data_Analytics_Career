#  Data Analysis Foundation & Tools

> [!INFO] Summary
> 
> This module establishes the core framework for the entire Professional Certificate, distinguishing between the **Data Lifecycle** (how data is managed) and the **Data Analysis Process** (how analysts solve problems) and Key **Data Analysis Tools**.

## 1. The Data Lifecycle vs. The Analysis Process

It is vital to distinguish between these two concepts:

- **Data Lifecycle:** The stages data goes through from creation to destruction.
    
- **Data Analysis Process:** The 6-step framework used to derive insights and solve business problems.
    

### Stages of the Data Lifecycle

- **Plan:** Define data needs, management, and responsibilities.
    
- **Capture:** Collect data from various sources.
    
- **Manage:** Store and maintain data securely.
    
- **Analyze:** Use data to support business goals.
    
- **Archive:** Keep relevant data for future reference.
    
- **Destroy:** Delete data securely.
    

## 2. The 6 Phases of Data Analysis

Each course in this program maps to one of these phases.

|**Phase**|**Core Objective**|
|---|---|
|**1. Ask**|Define the problem and confirm stakeholder expectations.|
|**2. Prepare**|Collect and store data for analysis.|
|**3. Process**|Clean and transform data to ensure integrity.|
|**4. Analyze**|Use tools to draw conclusions and make predictions.|
|**5. Share**|Visualize and communicate results for decision-making.|
|**6. Act**|Apply insights to solve the original problem.|

> [!EXAMPLE] Case Study: Employee Engagement Survey
> 
> The speaker uses this survey to show the lifecycle in action: from **Asking** what leaders need to know, to **Processing** the feedback for errors, and finally **Acting** by implementing organizational and team-level interventions.

> [!WARNING] The Risk of Skipping Steps
> 
> The process is rigorous and sequential. Skipping steps (e.g., jumping from "Ask" to "Analyze") leads to failed insights.

## 3. The Data Analyst Toolbox

### Primary Tool Categories

- **Spreadsheets:** Digital sheets like Excel/Google Sheets for small-to-medium datasets.
    
- **Query Languages (SQL):** Used to manipulate and retrieve data from databases (e.g., MySQL, BigQuery).
    
- **Visualization Tools:** Used to communicate findings (e.g., Tableau, Looker).
    

### Tool Selection: Spreadsheets vs. Databases

|**Feature**|**Spreadsheets**|**Databases**|
|---|---|---|
|**Access**|Via software app|Via query language|
|**Structure**|Rows and columns|Rules and relationships|
|**Capacity**|Limited|Huge|
|**Entry**|Manual|Strict/Consistent|
|**Users**|Single-user|Multi-user|
|**Control**|User-controlled|DBMS-controlled|

## 4. Analyst Mindset: Golden Rules

1. **The "Handshake" Rule:** The **Process** phase is your introduction to the data. Use it to understand structure, quirks, and nuances.
    
2. **The "Storyteller" Rule:** You are an amplifier for the data. Let the facts speak; do not let your intuition bias the results.
    
3. **The "Act" Rule:** Analysis is just the bridge. The real value is created during the **Act** phase, where insights drive real-world change.
